Question 1: What is ASP.NET Core?
ASP.NET Core is a free and open-source web framework developed by Microsoft. It runs on any operating system (Windows, Linux, macOS) and is used to develop web applications, REST APIs, and other modern web services.

The term open source means it is free to use and its source code is publicly available.
Question 2: Difference between ASP.NET (Classic) and ASP.NET Core
ASP.NET (Classic) ASP.NET Core
Runs only on Windows Runs on Windows, Linux, macOS (Cross-platform)
Mostly closed source Open Source
Uses System.Web (monolithic pipeline) Uses lightweight, modular middleware pipeline
Web Forms, MVC, Web API are separate Unified programming model
Deployment: IIS only Deployment: Kestrel, IIS, Nginx, Docker, etc.
Question 3: What is MVC?
MVC (Model-View-Controller) is a design pattern used to separate application logic.
  • Model: Manages data and business logic
  • View: Represents the user interface (UI)
  • Controller: Handles user input and interacts with Model and View
Question 4: What is Middleware?
In ASP.NET Core, middleware is a software component that processes HTTP requests and responses. Middleware components are arranged in a sequence called a pipeline.

Each middleware can either handle the request or pass it to the next middleware.
Question 5: What is Dependency Injection and its Types?
Dependency Injection (DI) is a design pattern that helps in building loosely coupled, testable, and maintainable applications.
  • Constructor Injection (Most common)
  • Property / Setter Injection
  • Method Injection
Question 6: What is Kestrel Server?
Kestrel is a cross-platform web server for ASP.NET Core. It is the default web server used in ASP.NET Core applications.

It is enabled internally by: 
var builder = WebApplication.CreateBuilder(args);
app.Run();
Kestrel is used when hosting without IIS or behind IIS, Nginx, or Docker.
Question 7: What is Static in C#?
The static keyword in C# is used to define members that belong to the class rather than to an object.
  • Static Variable: One copy shared across the class
  • Static Method: Can be called without creating an object
  • Static Class: Cannot be instantiated
  • Static Constructor: Executes once before first use
